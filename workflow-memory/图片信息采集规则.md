# 工作流记忆：图片信息采集规则

## 日期
2025-08-11

## 规则
爬取网页时，如果图片本身包含有价值信息（如表格、流程图、数据图、操作截图等），需要额外采集。

## 判断标准
- **有价值的图片**：表格、流程图、数据图、操作步骤截图、重要示意图
- **无价值的图片**：装饰性图片、logo、背景图、分隔线

## 采集方式
1. **OCR文字**：Playwright会自动识别图片中的文字（已实现）
2. **下载图片**：如果图片内容复杂（如表格、流程图），后台知识卡片编辑器支持粘贴图片直链或网页 URL，下载后追加到 `sourceExcerpt`
3. **记录到sourceExcerpt**：图片会以 `![原文图片](...)` 的本地图片引用保存；如果图中事实要参与回答，必须再把核实后的关键信息写进 `body`

## 当前状态
- OCR文字：已自动采集
- 图片下载：已在 `/admin/knowledge` 卡片编辑器实现，图片存放在 `public/knowledge-images/`

## 相关文件
- 图片导入 API：`src/app/api/knowledge/source-images/route.ts`
- 原文摘录渲染：`src/components/knowledge/source-excerpt.tsx`
- URL 抓取工具：`prisma/tools/fetch-yuque.ts`（输出图片候选，支持 `--download-images=1,3` 下载并生成 `sourceExcerpt` Markdown）
