# 问答 Agent——第一版最小落地方案

> 这份回答的是：**如果现在就开始做，第一版先做什么，先别做什么**。
> 一句话定位：先把“会理解、会反问、会补搜一次”做出来，别一口气把整套 agent 搞满。

---

## 1. 第一版的目标

第一版不是要“非常智能”，而是要做到三件事：

1. **能先理解问题**
2. **遇到太宽的问题会反问一句**
3. **必要时能补搜一次，而不是搜一次就死**

只要这三件事成立，它就已经从“搜索机”变成“问答 agent”的雏形了。

---

## 2. 第一版只做的三步

### 第 1 步：问题理解

先加一个轻量的理解层，输出这些东西就够了：

- 问题类型（事实 / 流程 / 建议 / 比较 / 选择）
- 用户真正想问什么（一句话）
- 第一轮搜索词
- 是否太宽、要不要反问

这一步先别追求很复杂，先能把“转专业哪个好”这种问题拆成“流程 / 建议 / 选择”就行。

---

### 第 2 步：反问一句

只拦明显太宽的问题。

例如：
- “你是想问转专业流程，还是想问不同专业怎么选？”
- “你更关心条件，还是更关心选哪个方向？”

规则：
- 一次只问一句
- 不是所有问题都问
- 只问最关键的歧义

这一步先别搞多轮对话状态机，太重。

---

### 第 3 步：两轮检索

第一版只做两轮：

- **第一轮**：按理解层给出的核心词搜
- **第二轮**：如果证据不够，再补一组词搜一次

第二轮的作用不是“无限追问”，而是给 agent 一个补救机会。

---

## 3. 第一版先不做什么

### 不做完整多轮对话

先别做“反问—用户回答—再反问—再答”的完整状态机。太重，第一版没必要。

### 不做复杂检索规划

先别搞三轮、四轮计划，也别把问题拆成很多子任务。两轮足够验证方向。

### 不做特别花的回答编排

先让回答“更像人说话”，但不用追求特别会聊。先比现在顺就行。

### 不动卡的存储结构

卡片还是原来的样子，不建树，不改标签体系。

---

## 4. 第一版的边界

这版要守住三条边界：

- **不胡说**：卡外知识不能补
- **不乱改卡**：存储结构先不碰
- **不做太大**：先验证 agent 流程是不是有效

---

## 5. 落地顺序

推荐顺序：

1. `question-understanding`
2. `clarification gate`
3. `two-round retrieval`
4. `answer composition`

顺序别反。

---

## 6. 这版成了以后，用户会感觉到什么

用户会明显感觉到：

- 它不再只盯着字面关键词
- 它会先想“你到底在问什么”
- 它会在不清楚时先问一句
- 它会补搜，不会一上来就放弃
- 它回答时会更像一个懂上下文的人

这就是第一版真正要交付的东西。
