# 云校园 · 产品文档

> 这份文档只回答两件事：**做什么** 和 **不做什么**。
> 不写技术分工、不写任务排期、不写代码组织——这些另有其他文档承接。

---

## 1. 一句话定位

**云校园是面向 NJU 本科生的学习生产力工具，把"讨论"、"读书"、"作业"、"日常计划"四件分散的事，收拢到同一个工作台里，由 AI 在每一件事里做合适的协助。**

- 主语是"工具"。不是社区、不是网课、不是 AI 助手。
- "生产力"的衡量是单位时间内"读懂多少 / 按时交多少 / 想清楚多少"，不是"日活时长"或"互动量"。
- AI 是工具的一部分，**不是产品本体**。AI 介入的姿态、边界、可控性是产品要负责的事，不能放任。

---

## 2. 目标用户

**严格限定：南京大学本科生**。

- 范围内验证完了再考虑往外扩。
- 跨校、研究生、跨国院校等都不在一期范围。
- 用户不是"想象中的某类大学生"，而是**作者自己 + 作者拉得到的同学**。下面所有功能描述，都以"我自己作为唯一/首批用户"做锚点。

---

## 3. 用户场景

三个场景方向，每个都对应产品里的一类核心能力。

### 场景 A：复习一段没听懂的内容
现状：翻 PPT、问同学、搜知乎，三个地方来回跳，最后还是糊涂。
云校园：把课程教材/讲义传进来，用**针对性问答**模式提问，AI 基于资料给带页码引用的回答；问答时可以自由切换"直接给答案 / 苏格拉底式 / 先诊断后补缺"三种姿态。

### 场景 B：作业越堆越多，差点忘掉
现状：DDL 散在课程网站、微信群、邮件、口头提醒、纸条里。
云校园：把这些散乱来源（粘贴文本、上课录音、记录本片段、截图）丢进**作业看板**，AI 抽取候选项，用户一次性审查确认入库。临近自动提醒，点一下直达提交链接。

### 场景 C：写完一篇有用的回帖，不想让它沉底
现状：好内容沉到第 200 楼，下次有人提同样问题还得再答一遍。
云校园：高质量回帖被**提名为论坛精华**，进入精华库。精华库不参与 AI 问答（论坛与教材问答各走各的），但作为论坛侧的可检索沉淀存在。

### 没单独写场景的：整章学习、ToDo
- **整章学习**是教材模块下的另一种模式（详见 §4.3），用户主动开启时才进入。
- **ToDo** 是非作业事项的工作台（详见 §4.5），按用户自己的节奏使用。
两者不属于"散乱痛点"，所以不放在场景里，而是作为"工具按需打开"。

## 4. 功能模块

云校园由 **5 个模块**组成。每个模块独立有用，组合更强。

模块清单：① 论坛与精华库 ② 教材问答与整章学习（两种模式同模块） ③ 作业看板 ④ ToDo ⑤ Hub。

### 4.1 论坛与精华库

**4.1.1 论坛**
分区 / 帖子 / 回复 / 标签 / 收藏 / 置顶 / 管理。NJU 邮箱注册 + 显示昵称（实名门槛见 §8）。

**4.1.2 精华库（论坛沉淀）**
- 优质回帖可被"提名为论坛精华"，进入一个**独立的论坛精华库**。
- 每条精华以"知识卡片"形态存在，可点回原帖。
- 精华库由 §4.1.3 的"问问学长 Agent"使用，不被教材模块的 agent 引用。

**4.1.3 问问学长 Agent（论坛精华问答）**
- 论坛模块下一个独立页（暂名"问问学长"）。
- 用户提问，agent 在**论坛精华库内**检索相关知识卡片，给带引用的回答，引用可点回原卡片/原帖。
- **依据源严格限定在精华库**——不引教材、不引整章学习内容。与教材问答 agent 完全并列。
- **问答姿态固定为"直接答"**。这个场景的本质是"转述同学经验"，不需要苏格拉底式引导。
- **跨源处理**：用户问的问题精华库里没人讨论过时，agent 用一般知识回答，但**必须明确标注**"这是基于一般知识的回答，精华库中未找到同学讨论"，避免让用户误以为有人讨论过。
- 与场景 A（教材问答）的边界：教材问答回答"书上怎么说"，问问学长回答"同学怎么解"。两者各引各的源。

**不做**：私信、群聊、动态流（朋友圈式）、表情包大战。

### 4.2 教材模块（针对性问答 / 整章学习 双模式）

教材模块下有两个模式，**默认是针对性问答**，**用户在教材界面点"开启整章学习"会切到一个新对话**。两种模式共享同一份资料（用户上传的教材 / 平台内置的资料），但是不同的对话流、不同的交互形态。

**4.2.1 针对性问答（默认模式）**
- 用户上传教材或选用平台内置资料，对该资料发起问答。
- AI 基于资料给带引用的回答（页码 / 章节锚点）。
- **三种问答姿态用户可切换**：
  - **直接给答案**：把答案讲清楚就停。
  - **苏格拉底式**：尽量不给答案，连续提问引导用户自己推。
  - **先诊断后补缺**：先让用户说"自己已知什么 / 想到哪一步"，AI 只补缺、纠错。
- **无依据拒答**：检索不到足够依据时，AI 必须明确说"资料里没有相关内容"，不允许补全式编造。

**4.2.2 整章学习模式（用户主动开启）**
专为"系统性学完一章"设计的工作流，按 4 个阶段走：

| 阶段 | 关键画面 / 行为 |
|---|---|
| **01 进入** | 选定一章后**直接展示该章节思维导图 + 子主题列表**。用户挑切入点。不问背景、不强制诊断。 |
| **02 子主题展开** | 点进一个子主题（如 "5.2 连续函数的性质"）：**左侧原文节选**，**右侧 AI 对话区**。AI 主动以"循循善诱、易理解"的口语方式讲该子主题的**起因 / 定义 / 理解方式**。用户可随时插问。 |
| **03 检查与推进** | 不强制检测、不强制门槛。AI 主动提供 **"隐藏深问"邀请**——业内人忍不住问、外行不一定问得出口的那种深入问题（如"为什么最值定理要要求闭区间"）。用户感兴趣才点开。 |
| **04 关闭与回访** | 本章结束（用户主动结束 / 自然学完）时产出**本章总结卡片**：思维导图 + 关键推导链 + 用户问过的深问清单。该卡片归档到该教材下。下次进入教材时，**章节列表显示进度**——已学 / 未学 / 已学但有未点开的深问。 |

**模式切换规则**：
- 在教材问答里点"开启整章学习" → 选一章 → 进入整章学习模式（新会话）。
- 整章学习中遇到的具体问题可以"提到针对性问答里继续问"。两个模式之间能互相跳，但是会话独立、互不污染。

**不做**：通用文档管理（Notion 那种）；多人协同编辑教材；跨教材联合检索（一期不做）。

### 4.3 作业看板

**主轴**：解决"忘 DDL"。
辅轴：解决"上下文丢了（在哪交、要交什么）"。
**不解决**：调度（先做哪个）。

**4.3.1 录入：多载体抽取**
作业从这些途径进来：
- 粘贴文本（课程大纲、群聊记录）
- 上传上课录音
- 上传记录本拍照
- 手动录入

AI 从载体里抽取候选作业项（标题、课程、截止时间、提交链接、备注）。

**4.3.2 用户审查确认**
抽取完成后弹出**审查确认面板**：
- 列出所有候选项。
- 用户勾选哪些真要、可改字段、可补充、确定后才入库。
- "审查机制" 是为了**对冲 AI 抽取准确率不够**的情形——产品不假装 AI 一定对，把最终决定权留给用户。

**4.3.3 看板与提醒**
- 看板按截止时间排序。
- 提醒在临近时触发（具体梯度策略一期不写死，先做"临近一档"，后续观察）。
- **每条作业带提交链接**，看板上一键直达。

**4.3.4 完成判定**
- 不验证是否真的提交（产品管不到也不该管）。
- **用户手点完成**即归档。

**明确不做**：
- 周期重复作业（先单次，攒够使用反馈再决定要不要加）。
- 跨课程"周负载"视图（先把单条作业的录入-提醒-跳转跑顺，再考虑视图）。
- 任何形式的平台爬取（合规风险，与定位无关）。

### 4.4 ToDo

**ToDo 接的是"不属于作业的任意计划事项"**。

**4.4.1 进入：空白页 + 拖拽起步**
打开就是一张可自由拖拽内容的画布。

**4.4.2 内置经典模板**
基于经典 ToDo 方法论（GTD / 四象限 / 每日三事 / 番茄等），平台内置几套**可复用、自定义度高的事项框架**。用户从内置库拖一个上画布作为起点。

**4.4.3 对话式微调（带版本机制）**
- 用户拖了模板上来后，可以跟内置 AI Agent 对话，让 AI 帮忙改字段、改结构、改交互逻辑。
- **每次 AI 微调产出一个新版本**（V1 / V2 / V3 命名）。用户能：
  - 保存当前版本
  - 给版本命名（如"考前冲刺版"、"日常版"）
  - 回滚到任意历史版本
- 版本机制是**对冲"AI 改坏了不可逆"风险**的设计。AI 可改边界不靠规则约束，靠版本兜底。

**4.4.4 资料库（装扮素材）**
ToDo 工作页的"装扮素材"，包括：
- 计时页背景图
- 计时结束铃声
- 其他视觉/听觉素材

来源两种：
- 平台内置一批
- 用户自己下载并导入

导入的素材通过 AI Agent **接入用户已自定义好的页面里**——例如"把这张图设为我番茄钟的背景"、"把这段音频换成结束提示音"。

**ToDo 与作业看板的关系**：完全分开。**作业看板里**的是"必须按时交、有外部 DDL"的事；**ToDo 里**是"我自己想做、节奏自定"的事。两边数据不打通，避免概念混淆。

### 4.5 Hub

**Hub 是独立模块**。

**一期里只装"论坛精华库"的跨章节展示入口**。也就是说，论坛精华库本来在论坛内部有个入口，Hub 提供一个"独立的、跨论坛分区的"浏览视角，方便像翻杂志一样看高质量沉淀。

**未来 roadmap**：Hub 可能装其他"值得被看见的东西"——但**具体放什么还没想清楚**。云校园不会为了"看起来热闹"而装东西，没想清楚就空着。

**不做**：商业化交易、内容评分系统、订阅制。

## 5. 一期 MVP 范围

**P0（必须做完才能上线）**：4 件事。
1. **论坛**（含精华提名机制 + 问问学长 Agent；精华卡片量少时 agent 也上，回答会以"精华库内容较少 + 一般知识"形态呈现，不等量起步）
2. **教材模块**（针对性问答 + 整章学习两种模式都要有，不能只做其一）
3. **作业看板**（多载体抽取 + 审查确认 + 提醒 + 跳转 + 手动完成）
4. **基础 ToDo**（空白画布 + 内置模板拖拽 + 对话式微调 + 版本机制；资料库可以晚一拍）

**为什么 4 件全要**：场景 A B C 各对应一类不可替代能力。砍掉任一件，"学习生产力工具"就名不副实。Hub 不在 P0 是因为它的形态依赖论坛精华机制成熟。

**P0 不包含**：Hub、ToDo 资料库、教材跨章节联合检索、复杂提醒梯度。

**一期不做**：
- 移动端 App（响应式 Web 即可）
- 多教材联合检索
- 多用户协同编辑
- 任何支付/会员

## 6. 路线图

| 阶段 | 内容 | 验收信号 |
|---|---|---|
| **P0** | 论坛 + 教材双模式 + 作业看板 + 基础 ToDo | 我自己能用云校园复习一门课、管完一学期所有 DDL，比之前用 PPT+知乎+纸条更顺 |
| **P1** | Hub（先装论坛精华跨分区入口）+ ToDo 资料库 + 提醒梯度细化 + 作业抽取准确率优化 | 至少一个同学愿意稳定使用，并给出反馈 |
| **P2** | 视觉/交互打磨 + Hub 第二种内容（如果想清楚了） + 跨教材联合问答（如果有人需要） | 我能把整理的章节总结卡片分享给同学，并被使用 |

合规作业同步（ICS / 邮件 / 插件）**从路线图中删除**——一期不做、未来也不做。云校园的作业能力定位就是"用户主动录入 + AI 抽取辅助"。

每个阶段的"验收信号"是**真实使用的判断**，不是功能数清单。

## 7. 设计原则（产品红线）

任何模块都要遵守的产品红线。技术怎么实现是另一份文档。

### 7.1 引用可追溯
任何由 AI 在教材模块给出的回答，都必须能展开到具体来源（教材的页码 / 章节）。**没有来源 = 不输出**。这条优先级高于"回答漂亮"。

### 7.2 无依据拒答
教材问答中检索不到足够依据时，AI 必须明确说"资料里没有相关内容"，并提示用户上传相关资料或换一个提问角度。**不允许补全式编造**。

### 7.3 不混源问答
**每个 AI agent 各引各的源**，不在同一回答的依据里混合：
- **教材问答 agent**：只引该资料的教材。
- **整章学习 agent**：只引该章节资料。
- **问问学长 agent**：只引论坛精华库。
不混源是产品边界，不是技术细节——它决定了"AI 答的是什么源的内容"始终对用户可解释。
**例外**：精华问答 agent 在精华库无命中时可用一般知识回答，但必须明标"非精华库内容"。

### 7.4 一个模型通道，可换
AI 调用统一走 OpenAI 兼容协议（baseURL + apiKey 从 env 读）。换模型只改环境变量，不改代码。这是产品对"成本与可替换性"的承诺。

### 7.5 不做平台爬取
作业、通知、成绩等，**全部用户主动录入或 AI 从用户上传的载体里抽取**。这条不只是合规，也是边界——云校园不是教务的影子。

### 7.6 用户最终决定权
**任何 AI 自动产出（作业抽取、ToDo 微调、整章总结、思维导图）都必须给用户一个"审查/确认/回滚"通道**：
- 作业抽取 → 审查确认面板
- ToDo 微调 → 版本机制
- 思维导图 / 总结卡片 → 可编辑、可重新生成
"AI 不会犯错"是危险假设，产品按"AI 一定会出错"设计兜底。

### 7.7 单用户先成立
任何功能在"我一个人用"的场景下必须成立。多人协作是加分项，不是前提。

## 8. 非目标（明确不做）

每次有人（包括我自己）提议这些方向时，看看这一节再决定。

- **不做模拟登录抓取课程平台**。
- **不做开放域闲聊 AI**——会稀释"教材引用 / 学习导向"这个核心。
- **不做实时聊天/IM**——已有微信/QQ。
- **不做多模态创作工具**（写作、画图、视频生成都不进一期）。
- **不做学分/绩点计算**。
- **不做考试代答、刷分、查重对抗**——红线。
- **不做付费会员/订阅制**。
- **不做混源问答**——论坛精华和教材内容不能混在同一个回答的依据里。

## 9. 成功指标

这一节的目的：**给产品装一套"踩刹车"的标尺**。
不是 KPI，不是给老板看，不是融资材料。是**写给一年后的自己**——任何新功能、任何方向调整，先看看这三条会变好还是变坏。变坏就别干。

**一期，三条指标**：

1. **每周打开 ≥ 5 天，且至少 3 天用了"教材问答"或"整章学习"**。
   表达："这是个高频学习工具，不是偶尔翻翻的资料库。如果我一周开两次都嫌多，说明工具太重或不够顺手。"

2. **本学期至少一门课，我用云校园复习的时间 > 用其他工具复习的时间**。
   表达："这是个主力工具，不是辅助玩具。如果我自己复习还要回 PPT+知乎，说明产品没真正接住痛点。"

3. **作业看板上线后，没有因为忘 DDL 错过任何一次提交**。
   表达："这是个有具体痛点结果的产品，不是看起来酷。如果我用着用着还是忘交，那看板形态就要重做。"

如果三条达不到，先别叫别人来用。

## 10. 已知未决问题

留个口子记不确定的事，别假装都想清楚了。

**身份与社区**
- **论坛实名门槛**：是否仅 NJU 邮箱注册？是否需要绑定真实姓名？匿名/实名平衡点在哪。
- **精华审核机制**：谁能提名、谁审核、起初我一个人能不能撑下去。

**版权与合规**
- **教材与讲义版权**：用户自己上传供自己使用属于合理使用。但如果用户想把"基于教材产出的总结卡片"通过 Hub 分享，与原文版权关系如何？一期通过"Hub 只放论坛精华、不放教材衍生卡片"先回避。

**AI 能力与质量**
- **AI 调用成本**：每月上限多少？谁付（自己 / 用户 / 校方支持）？超额怎么办？P0 上线前要有数。
- **思维导图与章节拆解的准确性**：整章学习模式严重依赖这一步。AI 生成错了怎么办？是否需要"手动校正/重新生成"机制？P0 至少要有"重新生成"。
- **隐藏深问质量**：深问要既触及核心又不悖于教材立场。质量不够时如何兜底？是否需要内置"深问示例库"？

**模块边界**
- **作业抽取准确率**：多载体（文本/音频/照片）抽取准确率有多大？目前的解法是**用户审查面板**——抽取完让用户勾选/修改/确认才入库，把"AI 错"转成"用户可以改"。这是产品兜底，不是技术承诺。
- **ToDo 中 AI 可改动的边界**：理论上 AI 能改字段、能改结构、能改交互逻辑。改太狠了会不会把模板搞坏？目前的解法是**版本机制**——AI 每次微调产出新版本（V1/V2/V3），用户可命名、可回滚。"AI 改坏了"不再是不可逆事故。

## 附录 A：术语表

- **论坛精华库**：论坛内部独立的"优质回帖收藏馆"。每条以知识卡片形态存在，可点回原帖。被问问学长 Agent 引用，不被教材模块引用。
- **问问学长 Agent**：论坛模块下独立的 AI 问答入口，依据源严格限定在论坛精华库。问答姿态固定为"直接答"。精华库无命中时可用一般知识，但必须明标。
- **针对性问答**：教材模块默认模式。基于用户上传或平台内置资料的问答，可切换问答姿态（直接答 / 苏格拉底 / 先诊断后补缺）。
- **整章学习**：教材模块的另一种模式。按 思维导图 → 子主题展开 → 隐藏深问 → 总结卡片 四阶段，系统性走完一章。
- **隐藏深问**：整章学习中 AI 主动提供的、业内人忍不住问但外行不一定问得出口的深入问题。用户主动点开才展开。
- **总结卡片**：整章学习结束时产出的归档物，含思维导图、关键推导链、深问清单。归档到该教材下，章节列表显示进度。
- **多载体抽取**：作业看板的录入方式。用户提供文本、音频、照片等任意载体，AI 抽取候选作业。
- **审查确认面板**：作业抽取结果展示给用户的最终确认界面。AI 抽取后用户勾选 / 修改 / 确认，才入库。
- **版本机制**：ToDo 模块中，AI 每次对话式微调产出一个版本（V1/V2/V3）。用户可命名版本、回滚到任意历史版本。
- **不混源问答**：产品红线。论坛精华、教材问答、整章学习的依据来源严格分开，不在同一回答里混合引用。
